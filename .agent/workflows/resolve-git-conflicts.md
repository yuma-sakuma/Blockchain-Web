# Git Conflict & Stash Resolution Workflow

คุณสามารถเรียกใช้ workflow นี้เมื่อเจอปัญหา Merge Conflict ระหว่างการรวมโค้ด (Merge/Pull) หรือเจอปัญหา conflict ค้างจากการใช้ `git stash pop`

## ขั้นตอนที่ 1: ตรวจสอบสถานะของ Git (Check Status)
เริ่มต้นด้วยการเช็คสถานะว่าตอนนี้มีไฟล์ไหนบ้างที่กำลังติดเครื่องหมาย Conflict อยู่ (Unmerged paths)

// turbo
```bash
git status
```

## ขั้นตอนที่ 2: ค้นหาตำแหน่งที่เกิด Conflict
ใช้คำสั่งเพื่อค้นหา conflict markers (`<<<<<<<`, `=======`, `>>>>>>>`) ในโปรเจกต์ เพื่อดูว่ามีไฟล์ไหนและอยู่บรรทัดไหนบ้างอย่างรวดเร็ว

// turbo
```bash
git --no-pager grep -n "<<<<<<<"
```

## ขั้นตอนที่ 3: ทำการแก้ไข (Resolve Conflicts)
Agent ทำการใช้คำสั่งเปิดไฟล์ตามรายการที่พบในขั้นตอนที่ 2 และพิจารณาแก้ไขโดย:
- โค้ดระหว่าง `<<<<<<< HEAD` (หรือชื่อกิ่งปัจจุบัน) จนถึง `=======` คือ **โค้ดฝั่งของเรา (Current changes)**
- โค้ดระหว่าง `=======` จนถึง `>>>>>>> [branch/stash]` คือ **โค้ดที่ดึงเข้ามาใหม่ (Incoming changes)**

*ให้ทำความเข้าใจและลบเครื่องหมาย (`<<<<<<<`, `=======`, `>>>>>>>`) ออกทั้งหมด จากนั้นเลือกว่าจะใช้โค้ดฝั่งไหน หรือนำทั้งสองฝั่งมารวมกัน*

## ขั้นตอนที่ 4: ยืนยันการแก้ไขและลบ Stash (หากเกิดจาก Stash)
หลังจากที่แก้ไขไฟล์ทั้งหมดเรียบร้อยแล้ว ให้สั่งบันทึกการแก้ไข:

```bash
git add .
git commit -m "chore: resolve merge conflicts"
```

**หากปัญหานี้เกิดจากการใช้ `git stash pop`:**
ปรกติแล้วถ้าเกิด Conflict ตอน pop ตัว Stash จะยังไม่ถูกลบออกจาก list ให้เราตรวจสอบและลบทิ้งเมื่อแน่ใจว่าแก้เสร็จแล้ว:
```bash
git stash list
# เลือกลบ stash ที่แก้ไขเสร็จแล้ว ตัวอย่างคือ stash@{0}
git stash drop stash@{0}
```
