# Checklist: Frontend & Backend Connection

This checklist tracks the integration of the React frontend with the NestJS backend.

## 1. Backend Configuration (NestJS)
- [ ] **Enable CORS**: Allow requests from the frontend origin (e.g., `http://localhost:5173`).
- [ ] **Global API Prefix**: Set global prefix to `/api` in `main.ts`.
- [ ] **Status Endpoint**: Add a `@Get('status')` endpoint in `AppController` for health checks.
- [ ] **Database Verification**: Ensure the backend can connect to MariaDB before answering status checks.

## 2. Frontend Configuration (React)
- [ ] **API Client Setup**: Install and configure `axios` or use `fetch` with a base instance.
- [ ] **Environment Variables**: Define `VITE_API_URL` for flexible configuration.
- [ ] **API Service Layer**: Create a service file to handle backend requests.
- [ ] **Connection Indicator**: Implement a UI component in the sidebar to show connection state.

## 3. Integration & Testing
- [ ] **Cross-Origin Test**: Verify the frontend can reach `/api/status` without CORS errors.
- [ ] **End-to-End Verification**: Ensure the "Backend Status" indicator turns green when the server is running.
- [ ] **Error Handling**: Gracefully handle cases where the backend is offline.
